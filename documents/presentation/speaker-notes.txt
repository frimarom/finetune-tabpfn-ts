Folie 11 Experimental Setup:
- For this we will now define our experimental Setup
- Our base code is from lennart puruckers repository Finetune TabPFNv2 which provides a good code base from which we will extend
- GIFT eval provides guidance via an API to load its datasets thus helping us to use these to create training, validation and test splits
- The featurization of the datasets is done with the code provided by the TabPFN-TS repository
- An important difference to other finetuning experiments is that we do not do a hyperparamter search but we create 54 Hyperparameter configs for each dataset
- we then sort by the best validation loss and take the 5 configurations with the best validation loss 
- we then evaluate only these 5 configurations with the corresponding test split from the dataset with the evaluation script from gift eval 
- The script provides many different Loss metrics. We will look at the Mean Absolute Scaled Error(MASE) and take the percentage difference between the finetuned model and the base model. Negative is better. Positive indicates a deterioration

Folie 15
- datasets only provide limited perspective
-> the datasets from gift eval only resemble some scenarios from real world datasets and thus provide only a limited perspective

- not all evaluation scores could be reproduced
-> we were not able to reproduce all evaluation scores of tabpfn ts on the gift eval benchmark. but this did not limited us in our results since we evaluated the base checkpoint on each dataset locally as well

- mismatch between validation loss and evaluation metric
-> validation loss was MSE and evaluation metric was MASE. this could be a big limitation but from experience we made with the various error metrics of the gift eval benchmark the MSE and the MASE were mostly proportional 

- TabPFN 2.5 was released during the thesis
-> TabPFN 2.5 and even 3.0 was released during the work on this thesis and presentation. the improvement of these models also lead to better tabpfn ts results that are better than our finetuned results

Folie 16
- improving validation/evaluation procedure
-> either adjust the validation procedure to the MASE oder the evaluation procedure to the MSE to have equal metrics

- dataset selection for multi-dataset finetuning
-> we selected random datasets from our three categories. making a curated list of datasets could lead to better finetuning results

- systematic hyperparameter search
-> a systematic hp search could also lead to a better understanding of how different parameters affect finetuning

- effect of newer model versions (TabPFN 2.5)
-> the configurations and results of TabPFN 2.5 and 3.0 could be used as a new baseline for future work

- sensitivity of prior-based continued pretraining
-> our continued pretraining did not show the expected generalization results. new time series data generating algorithms could lead to better finetuning results
