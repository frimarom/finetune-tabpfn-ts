\chapter{Introduction}

\section{Motivation}
Since the release of ChatGPT in 2022 and the surge of interest in artificial intelligence, machine learning and deep learning have become more accessible to the public \citep{mckinsey2025stateofai}.
Large Language Models are now known as being a gateway to machine learning and artificial intelligence.
But as the name suggests LLMs almost only excel on data in textual format.
However, what about tabular data?
Whether it is patient data in healthcare, sales data in finance or every other data which can be stored in an Excel-like schema, tabular data is one of the most common data formats used on a large scale \citep{van2024tabular}.
This type of data has been studied by researchers for a considerable amount of time but nobody has found a way to utilize deep learning for tabular data yet algorithms like gradient boost decision trees or random forest performed more efficiently and were used more commonly in this domain.
\newline
But with the release of TabPFN by \citet{hollmann2022tabpfn}, a transformer model pre-trained to solve artificially generated classification tasks from a synthetic prior, a new benchmark for tabular inference tasks has been set.
In a single forward pass, TabPFN is able to solve classification, regression, and inference tasks on tabular data with state-of-the-art results.
%\\
%Early 2025 TabPFN-TS was released.
%A method that maps time series data onto tabular data to achieve SOTA forecasting results with the already pre-trained TabPFN model.
\newline
There are two main ways to presumably enhance the performance of this specific model.
One is finetuning a pre-trained model and the other is continued pre-training of a pre-trained model.
The key difference between those two lies in the data the model is trained on.
By only using real-world datasets from a specific domain, like healthcare or finance, with finetuning one trains a pre-trained model to enhance its accuracy in this domain.
In continued pre-training, a specific synthetic prior is used to generate synthetical classification, regression, or inference tasks for the domain to further train and enhance the model.
\newline
When it comes to physics, finance, meteorology or any domain where predicting the future is crucial, time series forecasting models are indispensable.
A time series forecast is a forecast for the development of a system based on the current and prior observations of it e.g., predicting the weather based on past observations and historical data from the same date in previous years.
It utilizes the temporal dependencies and intervariable correlations within the observations.
\newline
There are many different approaches used in forecasting including Multi-Layer-Perceptron, a Recurrent Neural Network or even Graph Neural Networks.
In this thesis we will focus on transformer based time series forecasting as they have shown great modeling ability for long range dependencies and interactions in sequential data and thus are appealing to time series modeling.
\newline
Although TabPFN for time series already achieves top rank results in time series forecasting, the impact of finetuning or continued pretraining has not been studied and potential benefits remain uncovered.

\section{Exposé}
As \citet{hoo2025tables} showed, TabPFN can work with time-series as well, although it has not been trained with time-series data during pre-training, treating time series forecasting as a tabular regression problem.
The good performance in time-series data has two main causes: Preprocessing the timestamp feature of the data into automatic seasonal features, calendar features and running index and a pretrained model, which has a strong zero-shot performance on regression tasks. \newline
This thesis focuses on finetuning and continuing pretraining on TabPFN with time series data.
We will address the concept of the aforementioned methods and examine whether these methods lead to performance boosts in time series forcasting methods.
\newline
The questions we will address are:
\begin{enumerate}
    \item Can we finetune TabPFN for time series with time-series data (and does it improve the performance)?
    \item Can we further improve performance by finetuning on multiple datasets similar to RealTabPFN \citep{Hollmann_2025}
    \item Can we continue pre-training TabPFN for time series with a prior that generates time series data?
    \item Which one of the following models performs best on various high scale benchmarks: TabPFN-TS, finetuned TabPFN-TS, or continued pre-trained TabPFN-TS?
\end{enumerate}
Chapter 3 will focus on finetuning the model with datasets from the GIFT-EVAL benchmark as well as datasets from the autogluon/chronos-datasets repository \citep{aksu2024giftevalbenchmarkgeneraltime,ansari2024chronos} .
Most of the code used in this section was already provided by Lennart Purucker \url{https://github.com/LennartPurucker/finetune_tabpfn_v2}, but we had to adjust some of it to bring it up to date and adjust it to time series data.
To examine the presumable improvements we use the GIFT-EVAL evaluation code to calculate different types of errors before and after finetuning.
\newline
